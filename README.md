# mitumorikannri-VR3

作成した「見積管理-VR3」と「JIBUN-OSハブ」の連携アーキテクチャに沿って、実際の日常業務における使用手順（ユーザー視点のワークフロー）をまとめました。
自動化によって、手作業は**「ファイルの配置」**と**「最終確認」**のみに最小化されます。
### 📥 日常業務フロー (見積書・注文書の処理)
**ステップ1: ファイルのアップロード（トリガー）**
 * **あなたのアクション:** 取引先からメール等で届いた「注文書」や「見積書」のPDF・画像ファイルを、パソコンまたはスマホからGoogleドライブの**「未処理フォルダ」**に保存します。
 * *(システム側の動き): ファイルが追加されたことをGASが検知し、自動処理プロセスがバックグラウンドで起動します。*
**ステップ2: 自動処理の待機（数秒〜数十秒）**
 * **あなたのアクション:** 特にありません。コーヒーでも飲みながらお待ちください。
 * *(システム側の動き): GASがDocument AIを呼び出してOCR（テキスト抽出）を実行。抽出したデータをスプレッドシートの「部品DB」と照合し、過去の類似見積書とのマッチングロジックを走らせます。*
**ステップ3: Telegramでの通知受信**
 * **あなたのアクション:** スマホのTelegramアプリ（jibun_kaisha_v1_bot）に処理結果の通知が届くのを確認します。
 * **通知内容の例:**
   * 🟢 **成功:** 「✅ 注文書(A社)の処理完了。過去見積書(#123)と100%一致しました。スプレッドシートに転記済みです。[リンク]」
   * 🟡 **要確認:** 「⚠️ 注文書(B社)の品番に一部不一致があります（類似率85%）。確認をお願いします。[リンク]」
**ステップ4: 結果の確認と承認**
 * **あなたのアクション:** Telegramの通知にあるリンクをタップし、Googleスプレッドシートを開きます。抽出されたデータとマッチング結果をサッと確認し、もし文字化けやイレギュラーがあれば手動でサクッと修正します。
**ステップ5: 処理の完了（アーカイブ化）**
 * **あなたのアクション:** スプレッドシート上の「承認」チェックボックスにチェックを入れるか、Telegramボットに「/approve」等のコマンドを送信します。
 * *(システム側の動き): 対象のPDFファイルがドライブ内の「処理済みアーカイブフォルダ」へ自動的に移動し、フォルダ内が常に整理された状態に保たれます。*
### ⚙️ マスタデータのメンテナンス
システムの精度を保つための定期的なメンテナンス手順です。
 * **部品DBの更新:** 新しい部品が追加されたり、価格が変更になった場合は、**Googleスプレッドシートの「部品DB」シートを直接編集**します。GASは常にこの最新のシートを参照するため、システム側のコードを書き換える必要はありません。
 * **外部サービスとの連携 (オプション):** 新規の見積りが確定した際、JIBUN-OSハブ（n8n）を経由して、Googleカレンダーに納品スケジュールを自動登録したり、別のタスク管理ツールへタスクを発行することも可能です。
この業務フローを実現していくにあたり、まずはどの部分のコード構築から着手しましょうか？（例：「ドライブのファイル検知とOCR処理」から始めるか、「Telegramへの通知テスト」から始めるか等）
