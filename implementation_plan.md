# ノーコード管理基盤・権限ロール管理の実装計画

「設定マスタ」からUIや編集・閲覧権限まで動的に制御できるアーキテクチャの構築プランです。

## User Review Required

> [!IMPORTANT]  
> 以下の事項について、方針に問題がないかご確認をお願いいたします。
> 既存の「システム設定」シートの形式を拡張し、C列以降へ権限表示のマトリクス表を追加します。

## Proposed Changes

### `01 config and setup.gs` (バックエンド/初期化・設定取得ロジック)

#### [MODIFY] `01 config and setup.gs`
1. **初期化/マイグレーション処理 (`_setupSettingsSheet`)**
   - 既存の「カテゴリ」「値」「説明」の３列に対し、C列〜G列（閲覧権限：営業課、設計課、資材購買課、品質管理、上司）、H列〜L列（編集権限：営業課、設計課、資材購買課、品質管理、上司）のカラムを追加生成します。
   - すでにシートが存在する場合でも、列が不足している場合は自動で列を追加し、チェックボックス（データ入力規則 `requireCheckbox()`）を付与するマイグレーションロジックを実装します。

2. **ユーザー権限判定 (`getUserRole`)**
   - `Session.getActiveUser().getEmail()` を使用。
   - 「カテゴリ＝ユーザー権限」の行として各メールアドレスを「値」へ、所属部署を「説明」等に登録できるよう初期データを用意。このリストからログインユーザーの権限（例：「営業課」）を判定します。

3. **設定取得API (`getMasterSettings`)**
   - シート全体を読み込み、①ステータスリスト、②部署ごとのカラム閲覧・編集権限のJSON、③ユーザー権限マッピングを含むオブジェクトを返却する関数を作成します。

### `03_webapp_ap.gs` (バックエンド/APIルーティング)

#### [MODIFY] `03_webapp_ap.gs`
1. `handleApiRequest` 内に分岐を追加:
   - `case 'getMasterSettings': return getMasterSettings();`
   - `case 'getUserRole': return getUserRole();`
2. `getAllMgmtData` などで不要データ（非表示）がすでに弾かれているか、また権限に応じて送信前にマスク（`''`などで塗りつぶし）するべきかを検討（今回はHTML側での表示制御をหลัก拠とします）。

### `Dashboard_js.html` / `Dashboard.html` (フロントエンド/UI動的生成)

#### [MODIFY] `Dashboard_js.html` & `Dashboard.html`
1. **初期化のブロック化**
   - `window.onload` (または `window.addEventListener('load', ...)` 内で一番最初に `getMasterSettings` と `getUserRole` を `Promise` 等で取得し、グローバル変数に格納するロジックに変更。
   
2. **プルダウンの生成とバリデーション**
   - ステータスの `<select>` タグ（`filter-status`、`edit-status` など）に対し、ハードコーディングを排除し取得した `settings.statuses` 配列を利用して `<option>` を動的に追加。

3. **リスト・詳細の権限制御（閲覧・編集）**
   - リスト画面（案件一覧・台帳テーブル）生成時、`columns` 定義と取得した「閲覧権限フラグ」を比較し、`<th>` と `<td>` の出力自体をスキップ（非表示）にします。
   - 詳細モーダル内においても、DOM出力時に「編集権限がない項目」は `disabled` 属性を付与するか、もしくは `input` タグではなく `<span>` 等による読み取り専用テキストとして出力します。

## Open Questions

> [!WARNING]  
> 以下の点について教えてください。
> 
> 1. **カラム名と実データの紐づけ**:
>    現在テーブルの列名は「発注書番号」「機種」「顧客名」等が存在しますが、「設定マスタ」に登録するカラム名はこれらUIの文字列と完全一致させる仕様でよろしいでしょうか？（例：`カテゴリ="カラム制限"`, `値="金額"`）
> 2. **権限がないと判定された未登録ユーザーの扱い**:
>    シートに登録されていないメールアドレスからのアクセスの場合は、「権限：未設定」としてどの列も表示させない、または最低限「一般表示（ゲスト）」などのデフォルト権限を持たせますか？
